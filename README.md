Predicción de Accidentes e Incidentes de Buses en Londres

Este proyecto fue desarrollado con el objetivo de analizar accidentes e incidentes ocurridos en el sistema de transporte público de Londres utilizando técnicas de análisis de datos y Machine Learning. 
A partir de un dataset público que contiene información sobre condiciones climáticas, experiencia del conductor, horarios, tipo de víctima, carga de pasajeros y causas de accidentes, se buscó identificar patrones relevantes y construir modelos predictivos capaces de anticipar distintos tipos de incidentes.
El trabajo se enfoca principalmente en la clasificación de causas de accidentes y tipos de incidentes dentro de buses urbanos. Para ello se realizó un proceso completo de limpieza, transformación y análisis exploratorio de datos, acompañado por visualizaciones estadísticas que permitieron detectar relaciones importantes entre variables operativas y los eventos registrados.
Durante el análisis exploratorio se observó que los incidentes más frecuentes corresponden a caídas y lesiones dentro de los buses, mientras que eventos como robos, vandalismo o fallas críticas presentan una ocurrencia considerablemente menor. 
También se identificaron patrones relacionados con el clima, la hora del día, el tipo de víctima y la experiencia del conductor.
A partir de estos datos se desarrollaron modelos supervisados de clasificación utilizando Random Forest y Logistic Regression. 
Para mejorar la calidad del modelado se implementaron técnicas de ingeniería de atributos, creación de variables temporales derivadas de la fecha del incidente, codificación de variables categóricas mediante One-Hot Encoding y Ordinal Encoding, además de selección de variables con SelectKBest y mutual information.
El proyecto también incluye validación cruzada, análisis de métricas de desempeño y optimización de hiperparámetros mediante GridSearchCV. Los resultados obtenidos muestran que Random Forest logra un desempeño ligeramente superior respecto de Logistic Regression, aunque ambos modelos presentan limitaciones asociadas principalmente al fuerte desbalance entre clases y la complejidad del dataset.
Además del modelado predictivo, el trabajo permitió comprender mejor cómo influyen variables como el clima, el horario, el tipo de pasajero y la operación del servicio en la ocurrencia de accidentes. Esto demuestra el potencial del análisis de datos y Machine Learning como herramientas de apoyo para la toma de decisiones dentro de sistemas de transporte público urbano.
El proyecto fue desarrollado íntegramente en Python utilizando librerías como pandas, numpy, matplotlib, seaborn y scikit-learn, y fue ejecutado en Google Colab para garantizar reproducibilidad y facilidad de acceso.
